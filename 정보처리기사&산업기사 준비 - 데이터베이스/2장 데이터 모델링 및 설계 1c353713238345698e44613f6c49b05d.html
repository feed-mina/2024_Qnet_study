데이터모델의 개념
  1. 데이터 모델의 개념으로 적절한 것
  • 현실 세계의 데이터 구조를 컴퓨터 세계와 데이터 구조로 기술하는 개념적인 도구이다.
    • 데이터 모델은 현실 세계의 정보들을 컴퓨터에 표현하기 위해 단순화, 추상화하여 채계적으로 표현한 개념적 모형이다.
  • 컴퓨터 세계의 데이터 구조를 현실 세계의 데이터 구조로 기술하는 개념적인 도구
  • 현실 세계의 특정한 한 부분의 표현
  • 가상 세계의 데이터 구조를 현실 세계의 데이터 구조로 기술하는 개념적인 도구

  1. 데이터 모델의 구성요소가 아닌것
  • 논리적으로 표현된 데이터 구조
  • 구성 요소의 연산
  • 구성 요소의 제약조건
  • 물리적 저장 구조
    • 데이터 모델의 표현 해야 할 요소 : 논리적 구조(Structure), 연산(Operation), 제약조건(Constraint)

  1. 데이터베이스의 구성 요소 중 개체(Entity)에 대한 설명으로 적합하지 않은것
  • 속성들이 가질 수 있는 모든 값들의 집합이다.
    • └ 도메인 이라고 한다. (속성들이 가질 수 있는 모든 값들의 집합)
  • 데이터베이스에 표현하려고 하는 현실 세계의 대상체이다.
  • 유형, 무형의 정보로서 서로 연관된 몇 개의 속성으로 구성된다.
  • 파일의 레코드에 대응하는 것으로 어떤 정보를 제공하는 역할을 수행한다.

  1. What is the entity type definition correctly? 엔티티 타입을 맞게 정의 한것
  • a set of entities that have the same attributes 같은 속성을 갖는 개체들의 집합
  • a set of attributes that have the same entities 같은 개체를 갖는 속성들의 집합

  1. 데이터의 가장 작은 논리적 단위로서 파일 구조상의 데이터 항목 또는 데이터 필드에 해당하는것은
  • attribute

개체와 속성의 구분을 명확하게 하기. 개체는 한 개 이상의 속성으로 이루어진다. 그리고 속성은 데이터베이스 구성의 가장 작은 논리적인 단위이다.

  1. 다음 설명이 의미하는 A와 B의 관계는

An entity in an entity set A is associated with any number of entities in an entity set B. and an entity in B is associated with any number of entities in A

  • many to many

개체 집합 A속에 있는 하나의 개체는 개체 집합 B의 여러 개의 개체들과 연관되었으며 B의 한 개체가 A의 여러 개의 개체들과 연관되어 있는 관계

  1. 다음 문장의 빈 칸에 적합한 용어는?

개체 - 관계 모델

  1. 개체-관계 (E-R)모델에 대한 설명으로 잘못 된것은
  • E-R다이어그램ㅇ로 표현하며 P.Chen이 제안했다.
  • 일 대일 (1:1)관계 유형만을 표현 할 수 있다.
  • 개체 타입과 이들 간의 관계 타입을 이용해 현실 세계를 개념적으로 표현한 방법이다.
  • E-R다이어그램은 E-R모델을 그래프방식으로 표현한 것이다.

  1. 개체-관계 모델 (E-R)에 대한 설명으로 옳지 않은것은?
  • 개체와 개체 간의 관계를 도식화한다.
  • 개체 집합을 사작형으로 표시한다.
  • 관계를 다이아몬드로 표시한다.
  • 오너-맴버 (Owner-MembeR) 관계라고도 한다.

  1. 개체-관계(E-R)모델의 설명으로 옳지 않은 것은
  • 데이터를 인티티, 관계,속성으로 묘사한다.
  • E-R모델에서 엔티티는 실세계에서 개념적 또는 물리적으로 존재하는 실제 사용을 의미한다.
  • E-R 모델에서 속성은 엔티티를 묘사하는데 사용될 수 있는 특성을 의미한다.
  • E-R 모델에서 관계는 속성들에 대한 관계 표현이다.
    • E-R모델에서 관계는 속성들에 관한 관계표현 아니고 개체 간의 관계 표현이다.

  1. E-R 다이어그램의 구성 요소에 대한 표현이 틀리게 짝지어진것
  • 개체 집합 - 직사각형
  • 관계 집합 - 마름모꼴
  • 속성 - 원
  • 링크 - 화살표 (x)

  1. 개체-관계 모델(E-R모델)에 대한 설명으로 옳지 않은 것은?
  • 개체 타입과 관계 타입을 이용해서 현실 세계를 개념적으로 표현하는 방법이다.
  • E-R다이어그램은 E-R모델을 그래프 방식으로 표현한 것이다.
  • E-R 다이어그램의 다이아몬드 형태는 관계타입을 표현하며 연관된 개체 타입들을 링크로 연결한다.
  • 현실 세계의 자료가 데이터베이스로 표현될 수 있는 물리적 구조를 기술하는 것이다.
    • 현실 세계의 자료가 데이터베이스로 표현될 수 있는 물리적 구조를 기술하는 것은 논리적 데이터 모델이다.
    • E-R모델은 개념적 데이터 모델이다.

  1. P.Chen이 제안한것으로 현실 세계에 존재하는 객체들과 그들간의 관계를 사람이 이해하기 쉽게 표현한 모델은?
  • 개체-관계 (E-R) 모델
  • 관계 데이터 모델
  • 네트워크 데이터 모델
  • 계층 데이터 모델

망(네트워크) 데이터 모델

  1. 데이터 모델에 관한 설명 중 옳지 않은 것은?
  • 관계 데이터 모델은 개체와 관계 모두가 테이블로 표현된다.
  • 계층 데이터베이스는 부자 관계(Parent-Child Relationship)를 나타내는 트리 형태의 자료 구조로 표현된다.
  • 네트워크 데이터베이스는 오너-멤버 관계(Owner-Member Relationship)를 나타내는 트리 구조로 표현된다.
    • 네트워크 데이터베이스는 오너-멤버 관계를 나타내지만 트리 구조가 아닌 그래프 구조이다.
  • 데이터 모델은 데이터, 데이터 관계, 데이터의 의미 및 일관성 제약 조건 등을 기술하기 위한 개념적 도구들의 모임이다.

  1. 데이터베이스의 논리적 구조 표현을 그래프 형태로 표현하며 일대 다 (1:N) 관계에 연관된 레코드 타입들을 각각 오너(Owner), 멤버(Member)라 하고, 이들의 관계를 오너-멤버 관계라고도 일컫는 데이터 모델은?
  • 관계형 데이터 모델
  • 네트워크 데이터 모델
  • 계층적 데이터 모델
  • 객체지향적 데이터 모델

  1. 상위 하나의 레코드에 대하여 하위의 레코드가 복수 대응하고 하위 하나의 레코드에 대해서 상위 레코드도 복수 대응하는 데이터베이스 구조는?
  • 망 구조
    • 상하의 개념이 있으면서 서로 복수 대응 하는 구조는 망 (네트워크) 형 구조 하나 뿐이다.

  1. 논리적 데이터 모델 중 오너-멤버(Owner-Member) 관계를 가지며 CODASYL DBTG모델이라고 하는 것은?
  • 네트워크 데이터 모델

데이터베이스 설계

  1. 데이터베이스 설계 순서로 옳은 것은?
  • 요구 조건 분석 → 개념적 설계 → 논리적 설계 → 물리적 설계 → 구현

  1. 데이터베이스 설계에 있어 개념 스키마 모델링과 트랜잭션 모델링을 병행적으로 수행하는 단계는?
    • 개념적 설계
      • 개념적 설계는 개념 스키마, E-R모델, 논리적 설계는 특정 DBMS 모델 !

  1. 데이터베이스 설계 시 요구 분석 단계에서 나온 결과(명세)를 E-R다이어그램과 같은 DBMS에 독립적이고 고차원적인 표현 기법으로 기술하는 것을 무엇이라고 하는가?
  • 계층 데이터베이스 모델링(x)
  • 개념 스키마 모델링
    • 개념 스키마는 현실 세계에 대한 인식을 추상적 개념으로 표현하는 과정으로 특정 DBMS와는 무관하다. 그리고 E-R모델링 하면 개념 설계와 연관이 있다 !

  1. 데이터 베이스 설계 단계와 그 단계에서 수행되는 결과의 연결이 잘못 된것은?
  • 개념적 설계 단계 - 트랜잭션 모델링
  • 물리적 설계 단계 - 목표 DBMS에 맞는 물리적 구조 설계
  • 논리적 설계 단계 - 목표 DBMS에 독립적인 논리 스키마 설계
    • 논리적 설계 단계 - 목표 DBMS에 맞는, 즉 독립적이지 않고 종속적인 논리 스키마를 설계하는 단계이다.
  • 구현 단계 - 목표 DBMS DDL로 스키마 작성
