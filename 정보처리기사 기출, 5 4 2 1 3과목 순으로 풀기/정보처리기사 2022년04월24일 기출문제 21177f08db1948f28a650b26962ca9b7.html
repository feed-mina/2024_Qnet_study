  • 다형성이란 여러 가지 형태를 가지고 있다는 의미로 여러 형태를 받아들일 수 있는 특징을 말한다.

  • 메소드 오버로딩(Overloading)의 경우 매개 변수 타입은 동일하지만 메소드명을 다르게함으로써 구현, 구분 할 수 있다. (x)
    • 같은 이름의 메소드를 중복하여 정의하는것
    • 한 클래스 내에서 메서드의 이름은 동일하지만 매개변수의 수나 타입을 다르게 하여 재정의 하는것

  • 객체는 공통 속성을 공유하는 클래스들의 집합이다. (x)

  • 객체는 필요한 자료 구조와 이에 수행되는 함수들을 가진 하나의 독립된 존재이다.

  • 클래스 : 하나 이상의 유사한 객체들을 묶는다.

* 데이터베이스가 분산되어 있음을 사용자가 인식할 수 있도록 분산 투명성 (Distribution Transparency)를 배제해야 한다. (x) : 위치 투명성 (Location Transparency) 하드웨어와 소프트웨어의 물리적 위치를 사용자가 알 필요가 없다.

  • NFS (Network File System) : 네트워크 상에서 공유되는 파일 시스템, 다른 원격 호스트의 파일 시스템을 로컬 디스크에 접근하듯 간단하게 접근하여 자신의 디렉토리 처럼 사용 할 수 있다.

  • ADE Network (Activity On Edge Network ) 어떤 프로젝트를 마치기까지 수행되는 작업의 각 단계(상태)를 그래프의 정점(Vertex)으로 표현하고 작업 하나가 완료되어 다음 단계로 넘어가는 시간을 그래프의 간선 (Edge) 로 나타낸 방향 그래프
